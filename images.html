The Concept of Responsive Iamges

The concept of responsive images — images that work well on devices with widely differing screen sizes, resolutions, and other such features — and look at what tools HTML provides to help implement them. This helps to improve performance across different devices. Responsive images are just one part of responsive design, a future CSS topic for you to learn.

Changing the width and height

We can use the height and width attributes to resize an image. However, it's better to change the height and width in a photo editor, like Photoshop or Gimp, instead. This is because despite the change here, the original file may be large, and it's better to avoid transmitting large files over the internet when possible. We don't always know what kind of internet connection people may have. Keep files as small as possible.

Responsive design with images

we can use the srcset and sizes HTML attributes to list optional size images to display depending on screen size. This allows the browser to load different size images depending on the size of the screen. This has other bandwidth implications. Some people may be on mobile devices, and therefore may have slower internet connections. Using a smaller image file will cause the smaller image to load first, and therefore, faster on such devices. The src attribute is used here as a fallback device, in case srcset does not work.
